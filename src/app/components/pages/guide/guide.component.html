The goal of the exercise is to Build a Temperature monitor application using Angular and Boostrap by completing the following user stories:

User Story #1

As an anonymous end user, I want to be able to navigate to the "Records" page and see the "Add records" form.

Acceptance Criteria & Definition of Done
  • A new page component Records should be created in the /src/app/components/pages folder.
  • It should be linked to the navbar component src/app/components/common/navbar using the /records path and in the router located at /src/app.router.ts
  • The new page should include the jumbotron common component featuring the "Temperature records" title.
  • Below the jumbotron should be present a form with:
    • a label named "Temperature record";
    • a temperature text input field;
    • a submit button named "Add".
  • Using Boostrap to style the form is mandatory.
  • The form should be responsive.
  • The new page component must be unit tested for the presence of all elements described.

User Story #2

As an anonymous end user, I want to be able to enter maximum 8 temperature records, by entering a value and then clicking on the "Add" button.

Acceptance Criteria & Definition of Done
  • Create a new stateless service named "TemperatureMonitor" that adds records in an array.
  • When entering a record in the form and clicking the "Add" button the service is called to store the records.
  • The component should not allow more than 8 values to be added in the store.
  • Once the stored maximum values are reached the "Add" button should be disabled.
  • Add new unit tests to the page component spec file to cover the new functionality.
  • A unit test file should also be created for the new service to cover its functionality.

User Story #3

As an anonymous end user, I want to see the progress of added records.

Acceptance Criteria & Definition of Done
  • Add a new ng-bootstrap progress bar component on the page below the add button with a label of your choice above.
  • The progress bar should reflect the number of records added / maximum records (e.g. 2/8) and should be filled accordingly.
  • The color of the progress bar should be green, when 8 maximum records are hit it should become red.
  • Add new unit tests to the page component spec file to cover the new functionality.

User Story #4

As an anonymous end user, I want to be prevented from entering and invalid value such as strings in the "Temperature" input field.

Acceptance Criteria & Definition of Done
  • The component should allow the service to be called only with numeric values.
  • Using Bootstrap and Angular form validation the user should be informed when an error has occurred.
  • Here are some of the possible invalid actions:
    • The user tried to add a string instead of a number;
    • The user tried to add an empty value.
  • Add new unit tests to the page component spec file to cover the new functionality.

User Story #5

As an anonymous end user, I want to be able to see the median of the entered temperature records that I have entered.

Acceptance Criteria and Definition of Done
  • Create a new button named "Get Median" next to the "Add" button.
  • Using a ng-bootstrap popover display the current median value for the entered records when hovering the button.
  • The "TemperatureMonitor" service should be responsible for calculating the median.
  • The "Get Median" button should be disabled if no records are present in the store.
  • Add new unit tests to the page component spec file to cover the new functionality.
